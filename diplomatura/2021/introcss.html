Introducción al lenguaje CSS

El lenguaje CSS nos sirve para asignar propiedades a las etiquetas HTML, lo que nos permite separar «contenido» de «continente» en un documento concreto (por ejemplo, en una página web). De esta forma podemos darle estructura y diseño de una manera sencilla.

Definir un estilo en CSS

Un estilo CSS consta de dos partes principales: un selector y una o más declaraciones.

body { margin:1em; }

Donde body corresponde al selector y «margin:1em» es una declaración.

Una declaración CSS siempre termina con un punto y coma. Las clases están rodeadas por llaves. Para hacer que la CSS sea más legible se puede optar por poner las declaraciones de cada clase en diferentes líneas.

p { margin:0; text-align:justify; text-indent:1.5em; line-height:1.3em; }

p {

margin:0;

text-align:justify;

text-indent:1.5em;

line-height:1.3em; }

Para poder explicar la función de las clases o algún otro punto que se considere necesario, se utilizan los comentarios. La sintaxis de los comentarios CSS es muy diferente a la de los comentarios HTML, por lo que no deben confundirse:

style.css

/* esta línea es opcional */

Section0001.xhtml

<!--esta línea es opcional-->

Los delimitadores son, como se ve en el ejemplo: /* COMENTARIO */.

Cuando hagamos nuestras clases personalizadas es siempre conveniente comentarlas, siendo aconsejable, a veces, anotar como debe ser llamada en el archivo .xhtml.

Cómo llamar un estilo desde HTML

Hay tres formas de insertar estilos en HTML:

  • Estilo en línea. Cuando las declaraciones se expresan directamente con la sentencia style dentro del cuerpo de un archivo .xhtml (en este caso las declaraciones no van encerradas entre comillas y ni tampoco entre llaves):

    <p style="color:sienna; margin-left:20em">Párrafo de color siena</p>

    NO puede utilizarse en el ebook

  • Hoja de estilos interna. Se define en la sección head de un archivo .xhtml mediante la etiqueta <style>. Aparecerá así:

    <head>

    <style>

    hr {color:sienna;}

    p {margin-left:20px;}

    body {background-image:url("images/back40.gif");}

    </style>

    </head>

    NO puede utilizarse en el ebook

  • Hoja de estilos externa. Todos los estilos van agrupados en un archivo externo a los archivos de texto, con extensión .css (en el ePub Base de nuestra comunidad se denomina style.css). Cada .xhtml debe enlazar a la hoja de estilos con la etiqueta <link>, dentro de la sección de la cabecera (<head>).

    <head>

    <title></title>

    <link href="../Styles/style.css" rel="stylesheet" type="text/css" />

    </head>

    NOTA: Es la única forma de utilizar los estilos en los eBook.

ePub Base: style.css

En la plantilla de ePubLibre, el archivo ePub Base, se incluye una hoja de estilos externa llamada style.css que se ubica dentro de la carpeta Styles. Contiene una serie de estilos predefinidos para agilizar nuestra tarea, que han sido probados en diversos lectores electrónicos para asegurar su compatibilidad y adecuada visualización.

Estructura

En el style.css verás que las declaraciones se agrupan en tres grandes apartados, separados por sus correspondientes comentarios:

/* ---- ESTILOS GLOBALES Y DE SECCIONES FIJAS (NO MODIFICABLES) ---- */

Que como indica el comentario, NO SON MODIFICABLES. Este bloque engloba los estilos de las etiquetas que forman el ePub «body, p, a, sub, sup…» y las clases creadas expresamente para dar formato a las secciones fijas del ePub Base. El código que ahí aparece no es modificable, en caso de que necesitemos modificar el comportamiento de alguno de esos elementos cuando aparecen en el cuerpo del libro, lo haremos usando clases personalizadas.

/* ---- ESTILOS GENERALES (ADAPTABLES EN CASOS ESPECIALES) ---- */

En este bloque encontramos los estilos de las etiquetas y clases que resultarán muy útiles para la maquetación de nuestros aportes pero que no tienen una incidencia directa o significativa en las secciones fijas del ePub Base. El código que contiene es modificable según nuestras necesidades aunque tendremos que apoyarnos en clases personalizadas si los cambios afectan a las secciones fijas del ePub Base (como cuando queremos cambiar el aspecto de los <h1>).

En el caso de <h1> son directamente modificables las siguientes propiedades: margin, padding, border, font-size, font-style, text-align, text-indent, text-decoration, font-family, font-weight y color.

Si queremos modificar otra propiedad de <h1>, debemos crear una clase personal y añadirla a la etiqueta (<h1 class="personal">) para evitar modificaciones en la página título.

/* ---- ESTILOS PERSONALES ---- */

Este apartado aparece como última línea de archivo, ya que aquí deberán ir tus propias declaraciones, y en el ePub Base inicial está vacío.

Es conveniente personalizar el ePub Base, incluyendo en el archivo style.css nuestras clases ya validadas, es decir clases de las que tengamos la certeza de que no crean problemas en ningún lector. En el foro hay un banco de pruebas para validar los códigos que se sospeche que pueden no funcionar en algún lector.

NOTA: Si usas una versión personalizada del ePub Base, no olvides borrar las clases no usadas antes de publicar el libro.

También se puede poner tu nombre en titulo.xhtml e info.xhtml, etc.

Aplicación de estilos

La utilización de las clases almacenadas en nuestra «hoja de estilos» se realiza de dos formas, según sea un selector de algún elemento de HTML o un selector de clase.

El estilo de un elemento de HTML (como <body> o <p>) se declara en el style.css para llamarlo desde un .xhtml:

style.css

body { margin:1em; }

Section0001.xhtml

<body>…</body>

La llamada a un selector de clase (como «.versalita») igualmente necesita declararse en el style.css para llamarlo desde un .xhtml:

style.css

.versalita { font-size:0.8em; } /*fijate en el punto que antecede al nombre*/

Section0001.xhtml

<span class="versalita">XXII</span>

Ejemplos

Veamos algunos ejemplos de cómo aplicar clases y los resultados que producen para que puedas familiarizarte con el código CSS.

Algunos editores prefieren eliminar el indentado al comienzo de cada capítulo o tras una pausa. Es suficiente aplicar el estilo «.asangre»:

<p class="asangre">Párrafo</p>

Párrafo (texto con identado 1 em por defecto)

Párrafo (asangre)

Para indicar una pausa o cambio de tema podemos usar el estilo «.salto25» (usado para saltos de escena) o «.salto10» (usado para saltos en cartas, sinopsis, poemas). Los párrafos no se deben separar o «espaciar» sistemáticamente usando saltos:

<p class="salto10">Párrafo</p>

Párrafo

Para destacar un bloque de texto, como una carta o un recorte de periódico es muy útil el estilo «.cita»:

<p class="cita">Carta</p>

Carta

Podemos combinar distintos estilos para conseguir distintos efectos. Aquí un ejemplo de la combinación de los estilos «.cita», «.centrado» y «.negrita»:

<p class="cita centrado negrita">Párrafo</p>

Párrafo

Para recrear un rótulo o placa está el estilo «.banner»:

<p class="banner">Rótulo</p>

Rótulo

Por defecto todos los párrafos aparecen justificados. Para alinear el texto disponemos de los estilos «.derecha», «.izquierda» y «.centrado». El estilo «.versalita» nos proporcionará ese tipo de letra:

<p class="derecha versalita">Párrafo</p>

Párrafo

Estos son solo algunos de los estilos añadidos por defecto. Te aconsejamos que eches un vistazo al archivo style.css del ePub Base para poder ir entendiendo su estructura, y poder definir más adelante estilos propios o añadir estilos recomendados por otros editores cuando lo precises.
